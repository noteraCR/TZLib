Задача :
В базе данных MS SQL Server есть продукты и категории. Одному продукту может соответствовать много категорий, в одной категории может быть много продуктов. Напишите SQL запрос для выбора всех пар «Имя продукта – Имя категории». Если у продукта нет категорий, то его имя все равно должно выводиться.
---------------------------

Будет два столбца: ProductName и CategoryName
Если одному продукту может соответствовать много категорий, то по принципам нормальных форм должна быть таблица ProductCategories.

Соответственно у таблицы Products будет столбец ProductCategoriesId.

Если в одной категории может быть много продуктов, то по принципам нормальных форм данные о продуктах в категориях не будут храниться.

Соответственно связь будет через такая. Мы возьмём Id продуктов. По ним найдём все соответствующие этим Id все ProductCategories по ProductCategories.ProductId. У ProductCategories будет также столбец CategoryId, по которому мы возьмём Category.Name.

Но может быть такое, что у продуктов в ProductCategories не найдётся ни одного ProductId, тогда в таком случае необходимо написать на SELECT в столбце CategoryName - "Без категории" с помощью COALESCE
---------------------------
Соответственно запрос.
---------------------------
SELECT
	P.ProductName AS Product_Name,
	COALESCE(C.CategoryName, 'Без категории') AS Category_Name
FROM
	Products P
LEFT JOIN
	ProductCategories PC ON P.ProductId = PC.ProductId
LEFT JOIN
	Categories C ON PC.CategoryId = C.CategoryId
ORDER BY
    P.ProductId;
---------------------------
